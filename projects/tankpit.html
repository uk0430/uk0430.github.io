What It Does

TankPit is a Discord bot for a gaming community built around a classic online tank game. The bot gives the community AI-powered tools to look up players, browse leaderboards, generate custom award banners, and ask natural language questions about the game — all powered by a local LLM running on my own machine.

The most interesting part is /ask — a hybrid RAG + SQL Q&A system. Users type questions in plain English and the bot routes them to either a SQLite database (for structured queries like leaderboards and stats) or a vector search over 13,250 embedded player profiles (for open-ended questions).

/ask
Hybrid RAG + SQL AI Q&A. Routes structured questions to SQLite, open-ended to vector search over 13,250 embedded profiles.
/tank
Live player profile lookup via TankPit API. Renders a player card image with stats, rank, cups, awards, and faction color using PIL.
/leaderboard
Live leaderboard image with year, color, rank, and top-N filters. Renders sprite-based award icons from a spritesheet.
/award
Custom award banner generator. Multi-select awards, color, and size options. PIL renders and returns a PNG banner image.
/help
Embed listing all commands with usage examples. Ephemeral response so only the user sees it.

The RAG Pipeline

When a user asks /ask what awards does Grimlock have?, the bot first tries the SQL layer. If the question doesn't match a structured query pattern, it falls through to the RAG pipeline:

Both the embedding server (nomic-embed-text-v1.5) and the LLM (Mistral 7B Instruct) run locally via llama.cpp on two separate ports. This keeps everything private, free, and fast — no API calls to OpenAI or Anthropic.

The SQL Query Layer

Before hitting the LLM, every /ask question is routed through _db_query() — a keyword-based intent router that maps natural language patterns to SQLite queries against 13,250 player profiles. 

# Examples of SQL-routed questions:
"who has the most gold cups"  → SELECT name, gold_cups FROM tanks ORDER BY gold_cups DESC
"how many generals are there" → SELECT COUNT(*) FROM tanks WHERE rank='general'
"top ten blue overall"        → SELECT ... WHERE color='blue' ORDER BY leaderboard_rank
"list lightbulb award players" → SELECT name FROM tanks WHERE awards_json LIKE '%Lightbulb%'

# Each result includes:
# - Clickable profile links → tankpit.com/tanks/profile?tank_id=X
# - Faction color dots (🔵🔴🟠🟣) from player color field
# - Inline award emoji decoded from awards_json
# - Alt-tank deduplication via bf_tank_name field

The Data Pipeline

The knowledge base was built from scratch using three custom scrapers, all running async in Python against the TankPit API and website.

crawl_profiles.py
Scraped 13,250 player profiles from the TankPit API. Structured data: name, rank, color, country, stats, awards, cups, tank ID. Stored as .txt files in scraped/profiles/.
leaderboard_crawler.py
Crawled 1,326 pages of overall leaderboard + 4 years (2017–2023). Stores year-by-year rankings in SQLite leaderboard_history table.
bulletin_crawler.py
Scraped 14 years of daily bulletin board posts (2012–2026). 5,160 days of community discussions, tournament results, and player interactions.
rebuild_bible.py
Chunks all .txt files, embeds each chunk via nomic-embed (port 8081), and writes bible_chunks.json with text + embedding vectors. 13,250 chunks, 0 failures.

Full Tech Stack

Bot Framework
discord.py 2.6.4 — slash commands, ephemeral responses, views, select menus, modals, deferred interactions.
LLM Inference
llama.cpp running Mistral 7B Instruct Q4_K_M on port 8080. GPU offloading with -ngl 999, KV cache quantization for speed.
Embeddings
llama.cpp running nomic-embed-text-v1.5 Q4_K_M on port 8081. Cosine similarity via numpy for top-k chunk retrieval.
Database
SQLite (tankpit.db) — tanks table (13,250 rows), leaderboard_history table (year-by-year rankings), leaderboard_snapshots.
Image Generation
PIL (Pillow 12.1.1) — player cards, leaderboard images, award banners. MD5-keyed PNG cache. Runs in asyncio.to_thread to avoid blocking.
Environment
WSL2 / Ubuntu 24. Single-command launch via ~/start-tankpit.sh alias. Deployed on Railway.app (production, without local LLM).

What I Learned

Building this from scratch taught me more about production AI systems than any tutorial. The hardest parts were getting the RAG pipeline reliable — managing embedding cache TTLs, handling the async/sync boundary between discord.py and llama.cpp, and making the SQL intent router robust enough that users don't notice when it falls through to RAG.

The data pipeline was a real engineering problem. The scrapers needed rate limiting, resume capability (--resume flag), structured error handling, and proper async concurrency to collect 13,250 profiles + 5,160 bulletin board pages without getting rate-limited or losing progress on interruption.

The project shows that a NOC/IT background translates directly into building data pipelines and production systems — systematic thinking, fault tolerance, monitoring, and documentation are the same skills whether you're watching a satellite link or building a vector database.
