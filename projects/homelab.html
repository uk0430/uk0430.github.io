What It Does

This project turns a Raspberry Pi into an always-on network hub that mirrors the kind of monitoring and operations infrastructure I've run professionally in enterprise NOC environments. The goal was to stop relying on my desktop being on for DNS, monitoring, and services — and to have a real lab environment to practice and extend skills at home.

The Pi (hostname: uzi, 192.168.1.254) runs a 14-container Docker Compose stack including Pi-hole for DNS-level ad blocking, Prometheus + Grafana for metrics and dashboards, a Cowrie SSH honeypot on port 22, fail2ban for automated banning, Nextcloud for personal cloud storage, Netdata for real-time telemetry, Gitea for self-hosted Git, and Nginx Proxy Manager for reverse proxy and SSL. The TankPit Discord bot runs here too — migrated from Railway. Watchtower handles automatic container updates at 4am.

The Docker Stack

Pi-hole · :8053
Network-wide DNS sinkhole. 1,614,089 unique domains blocked across 10 blocklists including StevenBlack, AdGuard, OSINT, Hagezi Pro, and Hagezi TIF. Desktop and iPhone use Pi-hole as primary DNS. Unbound on :5335 as upstream recursive resolver (DNSSEC).
Grafana · :3001
Security NOC dashboard showing Cowrie attack data, fail2ban bans, nmap network scans, and full container performance metrics via cAdvisor and Node Exporter. Home dashboard: Docker & Host Monitoring.
Prometheus · :9091
Metrics backend scraping node-exporter (system metrics) and cAdvisor (container metrics) every 30s. 30-day retention. Powers all Grafana dashboards.
cAdvisor · :8090
Google's Container Advisor — exposes per-container CPU, memory, network, and disk I/O metrics to Prometheus. Gives visibility into every running container in real time.
Cowrie · :22
SSH/Telnet honeypot running on port 22. Logs every brute-force attempt, credential pair, and command run by attackers. Real SSH is on port 2022. fail2ban bans repeat offenders for 24h.
Nextcloud · :8080
Self-hosted personal cloud with MariaDB backend. Replaces OneDrive as primary file storage. Nightly rclone backup to OneDrive at 2am. Accessible on LAN and via Tailscale from iPhone.
Netdata · :19999
Real-time performance monitoring. Pi is the parent node — desktop streams metrics to it via the Netdata streaming protocol. Covers CPU, memory, disk I/O, network traffic, and per-process stats.
Gitea · :3000
Self-hosted Git server mirroring all active repos (tankpit-bot, portfolio, dotfiles). Push hooks dual-write to both GitHub and Gitea simultaneously. Offline backup in case of GitHub outage.
Nginx Proxy Manager · :80/:81
Reverse proxy and SSL termination for all Pi-hosted services. Routes portfolio.home → nginx-portfolio, manages Let's Encrypt cert renewal, and handles internal service URLs.
TankPit Bot · background
Discord bot migrated from Railway to Pi Docker. Runs python bot.py with SQLite DB and RAG bible mounted as volumes. Restart policy: unless-stopped.
Watchtower · background
Automatic Docker image updates at 4am daily. Checks for new image versions, pulls, and restarts containers with zero manual intervention. Keeps the stack patched and current.

Network Inventory & Segmentation

Part of this project was a full audit of the 192.168.1.0/24 subnet — identifying every device, classifying it as trusted, IoT, or unknown, and planning VLAN/SSID segmentation using the router's built-in IoT WiFi network.

Trusted Devices
Desktop (ASUS, .160), iPhone (.166), iPad (.184), laptop (.183), Mac Time Machine (.156). Full LAN access, Pi-hole DNS enforced.
IoT / Media (move to IoT SSID)
Smart TV (.168), Echo/Fire (.157, .245), Ring camera (.159), Roku (.155), smart diffuser (.151), Epson printer (.170), Android TV (.244).
Flagged / Quarantine
.243 — Sichuan AI-Link device. Unidentified manufacturer, no known owner, active on the network. Flagged for isolation pending identification.
Infrastructure
Pi at .254 (static, DHCP reserved). Verizon CR1000B router at .1. Linksys AP at .100. Three SSIDs: Primary, Guest, IoT.

Roadmap

SNMP + Grafana
Python SNMP poller targeting the Verizon router and any managed devices. Feed into existing Prometheus + Grafana stack. Mirror the SolarWinds NPM/NTA interface-level traffic view from production.
Suricata NIDS
Network intrusion detection system on the Pi. Passive traffic analysis on LAN to detect port scans, ARP spoofing, and known CVE signatures. Feed alerts into Grafana Security NOC.
IoT Segmentation
Migrate all IoT devices to the Verizon built-in IoT SSID. Reduces attack surface — compromised smart device can't reach desktop or Pi on the primary subnet.
Gotify Push Notifications
Self-hosted push notification server. Alert on Cowrie attack spikes, fail2ban bans, and nmap new device detection — delivered to iPhone via Tailscale without a cloud relay.
Identify .243 (Sichuan AI-Link)
Unknown device with a Chinese manufacturer active on the LAN. Flagged for physical identification or router-level isolation pending investigation.
Nextcloud Client Sync
Install Nextcloud desktop client on Windows and mobile app on iPhone. Migrate OneDrive files into the self-hosted instance to complete the personal cloud migration.

What I Learned

Running a homelab is the fastest way to understand why enterprise networking decisions get made. I locked down DNS on my home network the same week I was troubleshooting a Verizon router that doesn't allow custom DHCP DNS servers — which is exactly the kind of ISP-side constraint we dealt with at ViaPath when coordinating with LECs.

Docker Compose taught me that container orchestration at small scale is more about operational discipline than technical complexity. Watchtower auto-updates, Nginx Proxy Manager for SSL, and volume mounts for persistence are the same patterns that appear in Kubernetes at scale.

Identifying the Sichuan AI-Link device (.243) was a real moment — an unknown device with a Chinese manufacturer on a home LAN is exactly what Splunk SIEM alerts are designed to catch in production. The homelab makes the professional context click.
